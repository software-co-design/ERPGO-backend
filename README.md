# 3 需求分析

## 3.1软件功能需求分析

### 3.1.1功能概述

产品库存与销售系统应具备如下功能：

（1）入库管理功能：包括产品入库申请及审核、入库信息统计记录等

（2）出库管理功能：包括出货单申请及审核、出库信息统计记录等

（3）销售管理功能：包括销售订单申请及审核、订单与销售信息统计记录等

（4）财务管理功能：包括订单销售信息统计、收款记录统计记录、对账功能等

（5）基本信息管理：包括员工账户信息管理、产品信息管理、员工工作量统计、客户信息管理等。

### 3.1.4需求的动态模型

（1）产品入库活动图

产品入库活动由车间人员填写入库单开始，仓库管理审批入库单，如果审批不通过，退回入库单车间人员需要修改原有入库单信息重新申请入库，如果审批通过，由仓库管理安排车间人员进行产品入库。

![图片](https://uploader.shimo.im/f/XkC0AmjHFoBWrIsw.png!thumbnail?fileGuid=3ETSm0mDdoAxHc6f)

（2）确定销售订单活动图

销售订单由销售人员填写申请，由销售经理审核，如果销售订单审核不通过，将退回销售订单，需要申请人修改订单信息重新申请。如果销售经理审核销售订单通过，将在系统中锁定商品数量，同时生成发货单递交到仓库部门。

![图片](https://uploader.shimo.im/f/68DTlYAGoOpernFn.png!thumbnail?fileGuid=3ETSm0mDdoAxHc6f)

（3）销售发货活动图

销售发货活动由销售部门确定销售订单后开始，系统由销售订单生成发货单递交给仓库管理，由仓库管理根据发货单填写出库单进行商品出库，考虑到销售订单商品种类、数目较多，仓库管理可根据发货单分批次填写申请出库单，对于每一次出库单的填写申请，都需要由仓库经理审核，审核通过后仓库管理可安排发货，一次销售发货活动的完成是以一个销售订单的所有商品全部出库为标志的。
